## Academy Tracks

<CardGrid stagger>
  <Card title="JavaScript and TypeScript" icon="seti:javascript">
    Build the language base required for serious backend code: scope, async
    behavior, modules, typing, generics, runtime validation, and practical
    Node.js integration.
  </Card>
  <Card title="Node.js and Express" icon="seti:node">
    Move from runtime fundamentals into servers, routing, middleware,
    validation, security, response patterns, and maintainable service structure.
  </Card>
  <Card title="Data and Security" icon="database">
    Model relational and document data, reason about indexes and transactions,
    and implement authentication, authorization, secrets, and API hardening.
  </Card>
  <Card title="Production Engineering" icon="setting">
    Practice testing, debugging, performance profiling, caching, queues,
    observability, containers, deployment, and system design trade-offs.
  </Card>
</CardGrid>

## Curriculum Preview

<div class="academy-curriculum">
  <article class="academy-curriculum-card">
    <span>Phase 1 · Weeks 1–6</span>
    <strong>Foundation &amp; Runtime</strong>
    <p>Speak the language, understand the runtime, set up real projects.</p>
    <ul>
      <li>JavaScript: scope, closures, async, modules</li>
      <li>TypeScript: types, generics, narrowing, runtime validation</li>
      <li>Node.js: event loop, streams, buffers, process</li>
      <li>Tooling: npm, semver, scripts, lint, format</li>
    </ul>
  </article>
  <article class="academy-curriculum-card">
    <span>Phase 2 · Weeks 7–14</span>
    <strong>APIs, Data &amp; Security</strong>
    <p>Build REST APIs backed by real databases with real auth.</p>
    <ul>
      <li>Express: routing, middleware, validation, errors</li>
      <li>SQL &amp; NoSQL: schemas, indexes, transactions</li>
      <li>Auth: sessions, JWT, OAuth, RBAC/ABAC</li>
      <li>API hardening: secrets, headers, rate limits</li>
    </ul>
  </article>
  <article class="academy-curriculum-card">
    <span>Phase 3 · Weeks 15–22</span>
    <strong>Architecture, Quality &amp; Scale</strong>
    <p>Design for change, prove behavior, and stay fast under load.</p>
    <ul>
      <li>Layered &amp; clean architecture, modular monolith</li>
      <li>Testing pyramid, mocking strategy, contracts</li>
      <li>Profiling, caching, queues, advanced APIs</li>
      <li>WebSockets, SSE, GraphQL, gRPC</li>
    </ul>
  </article>
  <article class="academy-curriculum-card">
    <span>Phase 4 · Weeks 23–27</span>
    <strong>Delivery &amp; Career</strong>
    <p>Operate production services and present work like a senior.</p>
    <ul>
      <li>Docker, CI/CD, release flow, rollback</li>
      <li>Observability, SLOs, runbooks, on-call</li>
      <li>System design for microservices &amp; messaging</li>
      <li>Capstones, reviews, interview readiness</li>
    </ul>
  </article>
  <article class="academy-curriculum-card">
    <span>Labs</span>
    <strong>Hands-on by default</strong>
    <p>Every lesson pairs with a lab that has explicit success criteria.</p>
    <ul>
      <li>Runnable code, not pseudocode</li>
      <li>Debug-a-broken-service exercises</li>
      <li>Scenario-based &ldquo;what would you do&rdquo; drills</li>
      <li>Architecture sketches and reviews</li>
    </ul>
  </article>
  <article class="academy-curriculum-card">
    <span>Assessment</span>
    <strong>Portfolio + interviews</strong>
    <p>Graduate with a portfolio that survives a senior code review.</p>
    <ul>
      <li>4 capstone tracks with scope &amp; rubric</li>
      <li>Checkpoints after every track</li>
      <li>Interview drills per module</li>
      <li>Production case studies</li>
    </ul>
  </article>
</div>
