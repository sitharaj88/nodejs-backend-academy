Relational databases are strong when data has structure, relationships, and consistency requirements.

## Core Concepts

Students should understand:

- tables
- rows
- primary keys
- foreign keys
- one-to-one, one-to-many, and many-to-many relationships

## Normalization

Normalization reduces duplication and keeps relational data easier to maintain.

Students do not need to memorize theory-heavy forms first. They should learn the practical goal: avoid fragile duplication unless there is a strong reason not to.

## Joins

Joins are not advanced trivia. They are core relational thinking.

Teach when to use:

- inner joins
- left joins
- relation tables for many-to-many structures

## Common Mistakes

- duplicating relational data everywhere
- skipping foreign-key thinking
- designing tables around UI screens instead of domain relationships

## What To Remember

- relational modeling is about structure and relationships
- joins are part of normal backend work
- consistency often begins in the schema
