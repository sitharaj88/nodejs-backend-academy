Backend development does not stop when the code compiles. Real systems have to be built, deployed, observed, and operated.

This track turns Module 11 into a broader learning path.

## What You Will Learn

- environment configuration and container basics
- CI/CD and release flow thinking
- deployment discipline and operational visibility
- runtime operations and runbook awareness

## Recommended Order

1. [Environment, Config, Docker, and Containers](/learning/devops/environment-config-docker-containers/)
2. [CI, CD, Release Flow, and Deployment](/learning/devops/ci-cd-release-flow-and-deployment/)
3. [Observability, Runtime Operations, and Runbooks](/learning/devops/observability-runtime-operations-and-runbooks/)
4. [Modern DevOps Coverage](/learning/devops/modern-devops-coverage/)

## Goal

By the end of this track, learners should understand how backend systems move from local development into dependable runtime operation.
