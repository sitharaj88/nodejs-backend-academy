REST is not the only API style worth teaching.

## GraphQL

GraphQL is useful when clients need flexible data selection and schema-driven querying.

Students should also understand its challenges:

- resolver complexity
- N+1 problems
- authorization complexity

## gRPC

gRPC is useful in service-to-service environments that benefit from:

- strong contracts
- binary transport
- generated clients

## Advanced API Shape Thinking

The lesson is not “GraphQL versus REST” as ideology. The lesson is how communication shape changes:

- contracts
- validation
- performance
- tooling

## What To Remember

- advanced API styles solve specific problems
- each shape changes tradeoffs and operational complexity
- students should choose based on constraints, not hype
