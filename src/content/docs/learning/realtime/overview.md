Not every backend interaction is a simple request-response JSON API.

This track turns Module 10 into a broader learning path around real-time systems and advanced integration shapes.

## What You Will Learn

- WebSockets, Socket.IO, and server-sent events
- queues, jobs, webhooks, and event-driven coordination
- GraphQL, gRPC, and other advanced API shapes
- real-time tradeoffs around state, scaling, and delivery guarantees

## Recommended Order

1. [WebSockets, Socket.IO, and Server-Sent Events](/learning/realtime/websockets-socketio-and-sse/)
2. [Queues, Jobs, Webhooks, and Event-Driven Flows](/learning/realtime/queues-jobs-webhooks-and-event-driven-flows/)
3. [GraphQL, gRPC, and Advanced API Shapes](/learning/realtime/graphql-grpc-and-advanced-api-shapes/)
4. [Modern Real-Time Coverage](/learning/realtime/modern-realtime-coverage/)
5. [Labs, Projects, Interview Questions, and Case Studies](/learning/realtime/labs-projects-interview-case-studies/)

## Goal

By the end of this track, learners should understand when plain REST is enough and when more advanced communication patterns are justified.

## Practical Depth

This track now includes:

- real-time labs
- queue and webhook mini-projects
- interview comparison questions
- production case studies on delivery guarantees and scale
