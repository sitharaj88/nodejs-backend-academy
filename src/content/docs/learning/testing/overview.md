Testing and debugging are part of engineering, not cleanup work after coding.

This track turns Module 08 into a deeper learning path.

## What You Will Learn

- testing layers and when to use each
- mocking, fixtures, contracts, and realistic test data
- debugging flow, logs, and diagnostic thinking
- quality strategies that reduce regressions

## Recommended Order

1. [Unit, Integration, and API Testing](/learning/testing/unit-integration-api-testing/)
2. [Mocking, Fixtures, Contracts, and Test Data](/learning/testing/mocking-fixtures-contracts-test-data/)
3. [Debugging, Logging, and Diagnostics](/learning/testing/debugging-logging-diagnostics/)
4. [Modern Testing Coverage](/learning/testing/modern-testing-coverage/)

## Goal

By the end of this track, learners should know how to test behavior intentionally and debug failures with structure rather than guesswork.
