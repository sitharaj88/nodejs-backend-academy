title: Module 13 - Version Control, Collaboration, and Career Readiness
description: Git workflows, pull requests, code reviews, documentation, capstone polish, and interview preparation for backend roles.
---

**Duration:** 1 week

This final module turns technical work into professional presentation. Students learn how backend work is reviewed, documented, and discussed in team environments.

## What Learners Cover

- Git fundamentals including add, commit, branch, merge, and rebase
- pull requests, code reviews, and collaboration workflows
- conventional commits and release note awareness
- backend project README writing
- portfolio-ready capstone packaging
- interview preparation for Node.js backend roles

## Suggested Teaching Sequence

1. Review branch and commit discipline.
2. Run a pull request workflow on real code.
3. Improve project documentation and setup clarity.
4. Practice presenting a capstone and defending implementation choices.

## Live Examples

- a clean pull request with reviewable commit history
- README structure for backend onboarding
- architecture walkthrough for a capstone project

## Practical Labs

- prepare a capstone repository for public review
- perform peer code review on another student project
- write interview answers for event loop, auth, and architecture questions

## Exit Outcomes

- students can collaborate like junior backend engineers
- students finish with projects that are understandable to reviewers
- students can speak clearly about what they built and why
