The capstone phase should prove that learners can combine multiple modules into one coherent backend system. Each capstone below maps naturally to the brochure and to the curriculum sequence.

## Capstone 01: E-Commerce API

### Core Scope

- products, categories, inventory, carts, and orders
- authentication and role-based admin access
- payment workflow integration concepts
- pagination, validation, and transaction awareness

### What It Proves

- resource design across multiple related domains
- secure operational flows
- business logic separation and database reasoning

## Capstone 02: Social Media Backend

### Core Scope

- user profiles and follow relationships
- posts, comments, reactions, and feeds
- media upload handling
- notifications or real-time chat features

### What It Proves

- interactive product thinking
- performance and caching awareness for feed-style systems
- event-driven features with WebSockets or notification flows

## Capstone 03: SaaS Product Backend

### Core Scope

- user authentication and organization or team roles
- billing or subscription concepts
- reporting endpoints and admin actions
- clean architecture and modular services

### What It Proves

- business-oriented backend design
- separation between identity, tenancy, and reporting logic
- readiness for product-style backend work

## Capstone 04: Real-Time Support System

### Core Scope

- support tickets and role-based workflows
- live chat or socket-based updates
- agent and customer timelines
- webhook-ready event flows

### What It Proves

- real-time communication design
- workflow state management
- operational event thinking beyond CRUD

## Recommended Evaluation Rubric

| Area | What to check |
| --- | --- |
| API design | clear routes, consistent responses, good naming |
| Security | validation, auth, authorization, safe defaults |
| Architecture | sensible layers, reusable modules, maintainable boundaries |
| Data modeling | correct schema design, query shape, pagination, indexing awareness |
| Quality | tests, logs, error handling, review readiness |
| Delivery | README, setup flow, deployment, CI, and demo quality |

## Minimum Capstone Deliverables

- one repository with clean project structure
- one setup guide and API overview
- one deployed instance or deployable container
- one architecture explanation
- one demo walkthrough with known limitations and future improvements
